TOMY

Moteur rapide d’évaluation des couvertures

 

 

Caractéristiques techniques

Tomy est un moteur d’évaluation rapide des couvertures avec ou sans le procédé bi-source

Tomy calcul aussi la distribution des contacts

Tomy fonctionne soit avec un seul panel à la fois, soit avec la dll panelloader.dll qui pemret de partager les panels.

Il est capable de travailler sur des probabilités (méthode probabiliste) , ou sur des taux d’écoutes (méthode par comptage)

Il calcul la couverture et l’asymptote, le nombre de cas et la population, de la cible et de la niche

Il peut construire des sous-cibles à la volée ; dans ce cas il retourne aussi le GRP de la sous-cible

Il peut effectuer des ventilations.

Il peut utiliser le résultat d’une distribution de contacts pour construire une sous-cible complémentaire

Il peut travailler avec un panel de base et un sur-échantillon (Médiadata+Médiacabsat)

 

 

Il ne calcul pas une base de calcul en multi-région

Il ne calcul pas de consolidation en multi-sources

Il ne calcul pas des distributions croisées en multi-média

Il ne sait pas travailler simultanément sur des fichiers en taux ou en proba

Utilisation basic en moteur d’évaluation des couvertures

Ajout d’un membre privé dans un .h

Ctomy m_Tomy ;

 

           

// Etape 1

// Initialisation de paramètres

            CTomyParam param;

            param.m_CheminData="c:\\datatopchrono";

            param.m_TypeSupport=1;

            m_Tomy.SetParametres(&param);

 

// Etape 2

            // Initialisaton du panel

            m_Tomy.InitialisePanel("b100");

 

// Etape 3

            // Initialisation de la cible

            m_Tomy.InitialiseCible(3);

            // ou si l'on travaille avec des sous cibles

            // CTomy::InitialiseSousCible(int NrCible,int CritereFiltre,CWordArray * ModaliteFiltre)

 

            // A ce point on peut obtenir des informations sur la cible du Panel

            long nbcas=m_Tomy.GetNbCas();

            long population=m_Tomy.GetPopulation();

 

 

// Etape 4

            // Evaluation d’un plan

            m_Tomy.InitialiseSpot();

            m_Tomy.AddSpot(1 /*STATION*/,1/*JOUR*/,700/*HORAIRE*/,0.52/*GRP*/);

m_Tomy.AddSpot(1,1,800,0.64);

                                  

            // Lancement du calcul

            double couverture=m_Tomy.LanceEval();

 

 

Principe : l’ordre des étapes doit être respecté.

Il est possible de reprendre à partir de n’importe qu’elle étape. Dans ce cas il faut aussi exécuter les étapes suivante.

Utilisation de tomy avec le taux d’écoute

 

            Il faut que OPTION_DUREEECOUTE soit définit.

 

Il faut dans l’étape 2 initialiser le jour de début

Avec le premier jour de la campagne

M_Tomy. InitialiseJourReference(jour)

 

Dans l’étape 4, il faut donner dans le Addspot le jour relatif à cette date de référence

Utilisation d’une sous-cible

 

Dans l’étape 3 :

Utiliser InitialiseSousCible() à la place de InitialiseCible

 

De plus, le GRP est recalculé, il faut donc utiliser GetGRP(…) après avoir exécuter LanceEval(…) dans l’étape 4.

 

retour sur le haut de page

 

 

 

Calcul d’une ventilation

 

Dans l’étape 3 :

Utiliser InitialiseCibleVentile(…) à la place de InitialiseCible(…)

 

Ou dans l’étape 4, utiliser LanceEvalVentile(…) à la place de LanceEval(…)
