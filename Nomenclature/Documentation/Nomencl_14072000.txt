				STRUCTURE DES FICHIERS DECRIVANT L'ARBORESCENCE
				===============================================

Modification Juin/Juillet 2000
==============================
	1/Modif Nomenclature annuelle
	  Les arborescences produit et marque sont déplacées au niveau de répertoire année
	  Seul la nomenclature secteur est commune à toutes les années
	  
	  Fichier Annee.txt au niveau répertoire Nomenclature
	  #Nb Annee
	  Annee1
	  Annee2 etc....
	  On aura donc des répertoire c:\...\Nomenclature\Annee1\
	  			      c:\...\Nomenclature\Annee1\
	  			      
	  En résumé
	  C:\....\Nomenclature\Secteur.arb
	  		      \Acces.lck
	  		      \Annee.txt	 
	  		      \2000\Annonceur\ (avec tous les ss-répertoires)
	  			    produit.arb
	  			    marque.arb
			      \1999\Annonceur\ (avec tous les ss-répertoires)
	  			    produit.arb
	  			    marque.arb
	  		      \etc....
	  		      			     			  			    	
	2/Modif Annonceur
	Désormais les annonceurs pourront être lié à un secteur/famille
	De plus ils ne sont plus forcément lié à un groupe (inclure date exercice)
	On aura donc au total 3 possibilités pour les annonceurs
		Annonceur affilié à un groupe et ayant des produits
		Annonceur non affilié à un groupe et ayant des produits
		Annonceur isolé (ni groupe ni produit)
	



Fichier Produit.arb
===================

	Ligne 1 : commentaire
	Ligne 2 : Titre Arborescence Produit = #ANNONCEUR
	
	Niveau Groupe
	Ligne 3 : Nb Groupe Produit
	+ Lignes infos groupe (7 lignes composées de 	1 / Nom répertoire Groupe Produit
							2 / Nom Alias Groupe 
							3 / Code Groupe
							4 / Date exercice (format dd/mm/yyyy)
							5 / Type 6 pour Groupe
							6 / non utilisée
							7 / non utilisée
							8 / Nombre annonceurs
							
	Niveau annonceur (9 lignes)			1 / Nom répertoire Annonceur
							2 / Nom Alias Annonceur
							3 / Code Annonceur
							4 / Date Exercice Annonceur non lié groupe (dd/mm/yyyy)
							5 / Famille Annonceur
							6 / Secteur Annonceur
							7 / Type 5 pour Annonceur
							8 / non utilisée
							9 / non utilisée
							10/ Nombre de produits
							
	Niveau Produit	(9 lignes)			1 / Nom répertoire Produit
							2 / Nom Alias Produit
							3 / Code Produit
							4 / Marque du produit		
							5 / Famille du produit
							6 / Secteur du produit
							7 / Type 4 pour produit
							8 / non utilisée
							9 / non utilisée
	Fin de chaque produit #0

Fichier Secteur.arb
===================

	Ligne 1 : commentaire
	Ligne 2 : Titre Arborescence Secteur = #SECTEUR
	
	Niveau Famille
	Ligne 3 : Nb Famille
	+ Lignes infos groupe (8 lignes composées de 	1 / Nom Famille
							2 / Nb secteur pour cette famille
							
	Niveau Secteur
							1 / Nom Secteur
	
	Exemple:							
	// Descriptif arborescence SECTEUR
	#SECTEUR
	#2
	Famille1
	#2
	Secteur1
	Secteur2
	#0
	Famille2
	#3
	Secteur1bis
	Secteur2bis
	Secteur3bis
	#0

Fichier Marque.arb
==================

	Ligne 1 : commentaire
	Ligne 2 : Titre Arborescence Marque = MARQUE
	Ligne 3 : Nb marques
	suivi des nb marques
	
	Derniere ligne #0 : marque la fin des marques
	
	Exemple :
	// Description arborescence produit //
	MARQUE
	#2
	Marque1ter
	Marque2
	#0

					FICHIER ACCES.LOG
					=================
				
Description
===========
	Servira uniquement à pister les différentes nouvelles créations ou destructions (groupes,annonceurs,
	produits,secteurs) au niveau de l'administrateur de la nomenclature
	Egalement à gérer au niveau de la consolidation lors de la création des campagnes (on indiquera
	tout simplement la date et le nom de la nouvelle campagne créée)				
	
	Pour chacune de des modifications on indiquera :
	le nom de l'intervenant sur 10 car maxi : ADMIN pour administrateur ou NOM UTILISATEUR
	la date format dd/mm/yyyy + hh:mm suivi de la rubrique et de l'intitulé de la modification
	 
	exemple : 
	   ADMIN      22/05/2000 12:50 Groupe  : création TELECOM
	   ADMIN      22/05/2000 13:15 Produit : création NOKIA 6110
	   ADMIN      22/05/2000 13:16 Produit : suppression ERIKSON T10
	   MARIANNE   22/05/2000 15:10 Campagne: création DANONE
	   etc.....
	   
	La suppression de ce fichier sera à la charge de l'administrateur
	
	Eventuellemnt un message pourra apparaitre en début de lancement nomenclature si ce fichier
	dépasse une certaine taille
