Présentation

Le Data Fret est un dispositif permettant la mise à jour de données. L’origine de ces fichiers est un serveur FTP qui se trouve chez JFC. La destination est un poste de travail ou un serveur de fichier situé chez un client de JFC (voir schéma). La couche TCP/IP du poste en liaison directe avec le serveur JFC est nécessaire au progiciel (winsock.dll 16 bits). Le DATA FRET est compatible avec toutes les versions récentes de Windows (W3.11, W95 , NT3.51 et NT4). Les moyens du DATA FRET sont multiples :

Ø Utilisation de FTP pour copier des fichiers
Ø Exécution de programmes de mise à jour
Ø Vérification de situation (état des programmes, du disque...)
L’implantation de cet outil est souple, et permet de s’adapter à différents besoins :
Ø La mise à jour directe d’un poste (niveau1)
Ø La mise à jour directe d’un poste pilote, ce qui permet de vérifier sur un site la validité des données.
Ø La mise à jour d’un poste de niveau 2, une fois que les données ont été vérifiées et validées par le pilote.
Ø La mise à jour d’un serveur de fichier via le poste pilote.
Ø Le choix d’automatiser ou non certaines procédures (ex. : les tarifs)

Description

Les données Fret sont disposées sur le serveur FTP chez JFC. Elles sont organisées selon l'arborescence suivante :
  • Média
    • Tube
      • Wagon
Le répertoire média correspond au genre des mises à jour. (ex : Télé, Radio, Divers)
Le répertoire tube différencie le type des mises à jour.    (ex : Version, Tarif, Source, etc)
Le répertoire wagon possède les données.    (ex : p2, t2, s2)

Pour déclencher une mise à jour manuelle, le client dispose d'un raccourci pour chaque type de mise à jour.
Pour une mise à jour automatique, l'application se charge d'éxécuter la mise à jour configurée.
