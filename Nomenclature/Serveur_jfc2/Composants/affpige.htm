Introduction
    L'affinage est le processus de convertion de données d'un format à un autre. Il est utilisé par les entreprises pour transformer les fichiers de données qu'ils s'échangent sous le format qu'il souhaite, pour les intégrées ensuite dans leur propre application.
    L'affinage de la Pige s'effectue chaque semaine pour formater les données des fichiers ASCII que Secodip nous envoie dans une base de donnée Access (Microsoft), disposé chez JFC. Les deux fichiers sources sont disposées sur le serveur FTP de JFC, répertoire Secodip, par Secodip. Il y a un fichier Nomenclature (.lib) et un fichier DataSpots (.dat).

Présentation du programme
    AffPige effectue :

  • La mise à jour du fichier brut chez JFC depuis les fichiers sources (l'affinage). En simultanée, un fichier différentiel de mise à jour est crée.
  • La traduction des supports SECODIP->JFC
  • La création d'un fichier spot à partir du fichier brut
  • La maj des fichiers clés clients auto ou manuelle
  • La configuration des chemins des fichiers et répertoires liés à l'affinage
  • Le changement du mot de passe d'un fichier Access (Base de donnée)
La maj du fichier spot est faite depuis le programme PigeMaj.

Avant l'affinage

  1. Récupérer sur la machine FTP de JFC, répertoire Secodip (ftp/secodip), les fichiers sources compressés contenant les nouvelles données.
  2. Déposer ces fichiers dans le répertoire de traitement des sources (\\TRAITEMENT\Affpige\xxxxx\Source).
  3. Les décompacter dans ce répertoire.
  4. Supprimer les fichier(s) compressés.
  5. Renommer les fichiers sources en fichiers daté (99Sem01dat.dat et 99Sem01lib.dat par exemple) et les copier dans le répertoire de sauvegarde des sources (\\SERVEUR_JFC2\SourcesP\xxxxx) ainsi que le répertoire de transit (c:\art\xxxxx\PigeData)
  6. Faire un backup du fichier brut (\\TRAITEMENT\Affpige\pige_xxx.mdb) et du fichier spot (\\TRAITEMENT\Affpige\data\pigexxx.mdb) (format ZIP par exemple).

Après l'affinage
    Il ne reste plus qu'à renouveller le fichier spot. Un petit outils appelés PigeMaj s'occupe d'intégrer les nouvelles données du fichier différentiel (MajPigxxx.mdb) vers le fichier spot (Pigexxx.mdb).

Présentation du gestionnaire client

Le gestionnaire client permet :

  • D'ajouter, ou supprimer un fichier clé.
  • De modifier les informations liées à chacun des clients (société, adresses, code, dates, ...)
  • De mette à jour les groupes et les annonceurs.
  • De sauvegarder les modifications effectuées.
  • D'imprimer le carnet clients et la liste des groupes/annonceurs/produits d'un client.
Les données clients (familles, groupes, annonceurs et produits, ainsi que les dates de validités) sont fournis par SECODIP.

Et après tout çà?
    Lorsque les fichiers clés, ceux correspondant à la périodicité de la mise à jour (les hebdos, les mensuels), sont prêts, il faut les disposer dans le répertoire de transit \\TRAITEMENT\art\xxxxx. Et si ce n'est pas déja fait, il faut aussi disposer le fichier spot et le fichier différentiel de mise à jour dans le répertoire de transit \\TRAITEMENT\art\xxxxx\PigeData.
    Ces manipulations sont nécessaires pour la préparation de ces fichiers au DataFret.
